实验一：按键外部中断与非阻塞消抖

掌握 ESP32-C3 GPIO 输入配置、外部中断使用、非阻塞消抖实现

实验原理

机械按键存在抖动问题，直接读取会产生多次触发。本实验使用 millis() 实现非阻塞消抖， 中断服务函数只设置标志位，主循环处理业务逻辑。

硬件连接

KEY1 连接 GPIO20，KEY2 连接 GPIO21，使用下拉输入模式，按键按下时为高电平。

代码实现

// 按键中断配置
void key_init() {
    pinMode(KEY1_PIN, INPUT_PULLDOWN);
    pinMode(KEY2_PIN, INPUT_PULLDOWN);
    
    attachInterrupt(digitalPinToInterrupt(KEY1_PIN), key1_isr, RISING);
    attachInterrupt(digitalPinToInterrupt(KEY2_PIN), key2_isr, RISING);
}

// 中断服务函数
void IRAM_ATTR key1_isr() {
    key1_int_flag = true;
}

// 非阻塞消抖处理
void key_task() {
    if (key1_int_flag) {
        if (millis() - key1_last_time > DEBOUNCE_TIME) {
            if (digitalRead(KEY1_PIN) == HIGH) {
                key1_state = !key1_state;
            }
            key1_int_flag = false;
            key1_last_time = millis();
        }
    }
}

实验步骤

  1. 1
    配置按键引脚
    将 KEY1 连接到 GPIO20，KEY2 连接到 GPIO21
  2. 2
    编写中断服务函数
    在中断服务函数中设置标志位，避免在ISR中处理复杂逻辑
  3. 3
    实现非阻塞消抖
    使用 millis() 函数实现非阻塞延时消抖
  4. 4
    测试验证
    编译下载程序，打开串口监视器验证按键功能
💡 中断服务函数应尽量简短，避免使用 delay() 等阻塞函数，推荐使用 volatile 变量传递状态。

实验二：传感器数据采集

掌握 DS18B20 温度传感器、ADC 光照传感器的数据采集方法

实验原理

DS18B20 使用单总线协议通信，通过 1-Wire 接口读取温度数据。光照传感器通过 ADC 采集模拟电压值， 转换为光照强度（lux）。

硬件连接

DS18B20 连接 GPIO10，光敏电阻连接 GPIO1（ADC 输入）。

代码实现

// DS18B20 温度读取
float read_temperature() {
    sensors.requestTemperatures();
    return sensors.getTempCByIndex(0);
}

// ADC 光照读取
int read_light() {
    int raw = analogRead(LIGHT_PIN);
    float voltage = raw * (3.3 / 4095.0);
    int lux = map(raw, 0, 4095, 1000, 0);
    return lux;
}

// 传感器任务
void sensor_task() {
    if (millis() - sensor_last_time > SENSOR_INTERVAL) {
        temperature = read_temperature();
        light = read_light();
        sensor_last_time = millis();
    }
}

实验步骤

  1. 1
    连接传感器
    将 DS18B20 连接到 GPIO10，光敏电阻连接到 GPIO1
  2. 2
    安装库文件
    安装 OneWire 和 DallasTemperature 库用于 DS18B20
  3. 3
    编写采集函数
    实现温度和光照的读取函数
  4. 4
    测试验证
    观察串口输出的传感器数据，验证采集功能

实验三：自动控制逻辑实现

实现基于传感器数据的自动控制逻辑

控制逻辑

温度高于阈值时开启风扇，低于阈值-2°C时关闭；光照低于阈值时开启灯泡，高于阈值+10lux时关闭。

代码实现

// 自动控制任务
void auto_control_task() {
    // 温度控制（风扇）
    if (temperature >= temp_threshold) {
        relay_on(FAN_RELAY);
    } else if (temperature <= temp_threshold - 2) {
        relay_off(FAN_RELAY);
    }

    // 光照控制（灯泡）
    if (light <= light_threshold) {
        relay_on(LAMP_RELAY);
    } else if (light >= light_threshold + 10) {
        relay_off(LAMP_RELAY);
    }
}

// 继电器控制
void relay_on(int relay) {
    digitalWrite(relay, HIGH);
}

void relay_off(int relay) {
    digitalWrite(relay, LOW);
}

实验步骤

  1. 1
    设置阈值
    设置温度阈值和光照阈值
  2. 2
    实现控制逻辑
    编写基于传感器数据的自动控制代码
  3. 3
    测试验证
    调节温度和光照，验证继电器自动开关功能
⚠️ 设置适当的回差值可以防止继电器频繁切换，延长使用寿命。

实验四：MQTT 远程控制

实现 MQTT 协议通信，实现远程监控和控制

实验原理

MQTT 是一种轻量级的发布/订阅消息协议，适合物联网设备通信。设备通过 MQTT Broker 实现状态上报和远程控制。

代码实现

// MQTT 连接配置
void mqtt_init() {
    client.setServer(mqtt_broker, mqtt_port);
    client.setCallback(mqtt_callback);
}

// MQTT 回调函数
void mqtt_callback(char* topic, byte* payload, unsigned int length) {
    String msg = String((char*)payload, length);
    
    if (String(topic) == "chemctrl/" + device_id + "/command") {
        if (msg == "lamp_on") relay_on(LAMP_RELAY);
        if (msg == "lamp_off") relay_off(LAMP_RELAY);
        if (msg == "fan_on") relay_on(FAN_RELAY);
        if (msg == "fan_off") relay_off(FAN_RELAY);
    }
}

// 状态上报
void mqtt_report() {
    String json = "{\"temp\":" + String(temperature) + 
                  ",\"light\":" + String(light) + 
                  ",\"lamp\":" + String(lamp_state) + 
                  ",\"fan\":" + String(fan_state) + "}";
    client.publish("chemctrl/" + device_id + "/status", json.c_str());
}

实验步骤

  1. 1
    配置 MQTT Broker
    设置 MQTT Broker 地址、端口和认证信息
  2. 2
    实现回调函数
    编写 MQTT 消息回调函数处理远程命令
  3. 3
    实现状态上报
    定时向云端发送设备状态
  4. 4
    测试验证
